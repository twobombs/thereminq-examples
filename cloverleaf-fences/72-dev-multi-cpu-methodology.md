# Methodology

To simulate the ground state properties of the strongly correlated 72-qubit Heisenberg XX model—defined by the Hamiltonian $H = -\sum_{\langle i, j \rangle} (X_i X_j + Y_i Y_j)$—we employ a PyQrack-accelerated Holographic Distributed Engine. This architecture circumvents the exponential memory requirements of monolithic state-vector simulation by partitioning the global state into smaller, classically tractable tensor network fragments coupled via a parameterized entanglement bath. The methodology follows a rigorous hybrid quantum-classical pipeline.

### a) Lattice Topology and Circuit Fragmentation
The target system is a 72-qubit physical lattice arranged in a $6 \times 12$ geometric grid, as implemented via the `get_topology()` function. To enable distributed execution across finite memory resources, the global grid is systematically fragmented into a $2 \times 2$ arrangement of independent $3 \times 6$ planar patches (comprising 18 qubits each, mapped in `get_3x6_edges()`). This spatial decomposition mirrors the circuit cutting and entanglement forging techniques first formalized by IBM researchers [arXiv:2104.10220], wherein a large-scale bipartite quantum state is reconstructed from localized sub-circuit evaluations. The inter-patch boundary connections (`fence_edges`) are explicitly severed, transforming a 72-qubit global contraction problem into four parallelized 18-qubit evaluations orchestrated by the `HolographicDistributedEngine` class.

### b) Main Physical Ansatz and Local Entanglement
Within each isolated $3 \times 6$ patch, a variational quantum state $|\psi_{\text{local}}(\vec{\theta})\rangle$ is prepared using a hardware-efficient parameterized quantum circuit [arXiv:1704.05018]. As defined in the `isolated_holographic_worker()` function, the circuit depth begins with single-qubit rotations $R_x(\theta_1)$ and $R_y(\theta_2)$ applied across all physical qubits (`num_physical`), parameterized by a subset of the global physical vector `patch_params`. Following the single-qubit layer, localized nearest-neighbor entanglement is generated by sequentially applying CNOT (`apply_cx()`) operations across all `intra_edges`. This produces the core topological ansatz, capturing short-range correlations strictly within the boundaries of the 18-qubit local lattice.

### c) The Holographic Bath Layer and Ancilla Embedding
To compensate for the severed inter-patch entanglement, each 18-qubit patch is augmented with a localized set of parameterized ancilla qubits (`fence_qubits`), creating an "entanglement bath". This approach draws direct inspiration from tensor network holography and boundary-bulk dualities in quantum algorithms [arXiv:2106.12627]. For every severed edge in the original topology, a corresponding ancilla is introduced to the target patch. Within the `isolated_holographic_worker()`, both the boundary physical qubit and its corresponding ancilla are subjected to parameterized $R_y$ and $R_z$ rotations based on `boundary_params`, representing the environmental boundary vector. Finally, these boundary pairs are maximally entangled via pairwise CNOT gates, seamlessly embedding the mean-field entanglement spectrum of the external lattice into the local patch state.

### d) Local Observable Evaluation and Uncomputation
The energy contribution of each patch is determined by evaluating the expectation values of the intra-patch Hamiltonian terms, specifically $\langle X_i X_j \rangle$ and $\langle Y_i Y_j \rangle$. To maintain strict memory efficiency during these evaluations, the engine eschews costly quantum state cloning ($|\psi\rangle \to |\psi\rangle \otimes |\psi\rangle$). Instead, it employs unitary uncomputation—a foundational reversible computing technique adapted for modern variational algorithms [arXiv:2012.09265]. In `isolated_holographic_worker()`, after computing the probability marginals (`sim.prob()`) for a specific Pauli basis rotation (e.g., applying Hadamard gates for the $X$-basis and performing a deterministic CNOT check), the inverse unitary operations are immediately applied to return the subsystem to the unmeasured ansatz state. This sequential "compute-measure-uncompute" cycle preserves the continuous vector space without triggering garbage qubit proliferation.

### e) Global Energy Reconstruction via Boundary Marginals
With all isolated intra-patch energies computed, the engine classically reconstructs the inter-patch boundary interactions. By measuring local boundary marginals in both the physical ($X, Y$) and ancilla ($I, X, Y, Z$) bases and returning them as `boundary_expectations`, the distributed components are mathematically stitched back together along the severed edges. This exact classical summation over the reduced density matrices effectively contracts the Bell basis ($|\Phi^+\rangle$) across the boundary [arXiv:2104.10220]. The inter-patch energy is calculated within the orchestrator's `run()` method as $E_{\text{fence}} = -\frac{1}{4} \sum_P s_P \langle X_A \otimes X_B \rangle$ where $s_P$ are the corresponding `bell_signs` (yielding `x_term` and `y_term`), allowing the complete continuous energy landscape to be resolved without requiring physical multi-patch quantum links.

### f) Classical Orchestration and Multi-processing Oracle Comparison
To validate the entanglement embedding process, the distributed Holographic Engine (executed on isolated GPU workers via `mp.Pool`) is benchmarked against a `MonolithicCPUEngine` oracle. This oracle utilizes exact PyQrack Binary Decision Trees [arXiv:1512.03208] initialized with `is_binary_decision_tree=True`, forcing extreme process-level parallelization via `oracle_evaluation_chunk()` to deterministically evaluate the un-bathed 72-qubit ansatz exactly. The absolute difference between the exact 72-qubit state energy and the reconstructed 4-patch distributed energy provides the quantitative "Training Loss", measuring the fidelity to which the holographic bath parameters can mimic the true global lattice correlations.
